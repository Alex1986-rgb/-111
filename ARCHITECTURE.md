# Архитектура проекта TextFlow

Этот документ описывает внутреннее устройство приложения, принятые архитектурные решения и структуру данных.

## 1. Общий обзор
Приложение построено по принципу **Single Page Application (SPA)** на базе React 19. Основной упор сделан на высокую производительность, SEO-оптимизацию и удобство пользователя.

## 2. Управление состоянием (State Management)
Для управления глобальным состоянием используется **React Context API**. 
- Контекст `AppContext` хранит данные о:
  - Списке услуг и их актуальных ценах.
  - Активных и завершенных заказах.
  - Транзакциях.
  - Состоянии авторизации администратора.
  - Системных логах (для отладки и уведомлений).
- Данные синхронизируются с **LocalStorage**, что позволяет сохранять прогресс пользователя и настройки администратора после перезагрузки страницы.

## 3. Маршрутизация (Routing)
Используется кастомная система маршрутизации на основе `window.location.hash`. Это обеспечивает:
- Легкость реализации без внешних библиотек.
- Корректную работу на статических хостингах.
- Возможность прямой навигации по ссылкам (например, на конкретную статью блога).

## 4. SEO и Мета-данные
Для обеспечения высокого ранжирования в Яндекс и Google реализован компонент `SEOManager`.
- Динамически обновляет `document.title` и мета-теги при смене страниц.
- Генерирует и внедряет JSON-LD разметку (Schema.org) для статей и главной страницы.

## 5. Интеграция с AI
Взаимодействие с Google Gemini API происходит напрямую через клиентскую библиотеку `@google/genai`.
- **AIHelper**: Помощник для генерации идей.
- **TextAuditTool**: Инструмент для анализа текстов на SEO-показатели.
- Все запросы используют модель `gemini-3-flash-preview` для обеспечения минимальной задержки.

## 6. Массовые заказы и Автоматизация
Реализован модуль `BulkOrderView`, который позволяет:
- Загружать ТЗ из внешних файлов (.xlsx, .csv).
- Автоматически рассчитывать стоимость для больших объемов контента.
- Генерировать структуру данных для последующего API-импорта на сайты клиентов.

## 7. Стилизация
Используется **Tailwind CSS 4.0**.
- Применяется концепция "Glassmorphism" для создания современного, легкого интерфейса.
- Адаптивность реализована через стандартные брейкпоинты Tailwind (`sm`, `md`, `lg`, `xl`).

## 7. Безопасность
- Доступ к панели администратора защищен (имитация авторизации).
- Все вводы пользователя проходят валидацию в компонентах форм.
- API ключи вынесены в переменные окружения.
