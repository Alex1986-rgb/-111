
# 📄 TextFlow: Техническая документация v2.0 (SaaS MVP)

Этот документ описывает текущую архитектуру и задачи по превращению прототипа в промышленное решение.

---

## 🏗 1. Текущая архитектура (Pseudo-Backend)

Приложение спроектировано по принципу **Separation of Concerns (SoC)**, что позволяет бесшовно мигрировать на Next.js.

- **Frontend Core**: React 19 + Context API (App.tsx работает как глобальный Store).
- **Service Layer**: Логика работы с AI и платежами вынесена в компоненты с имитацией API-задержек.
- **Routing Engine**: Кастомный Hash-router, обеспечивающий уникальные URL для SEO-контента и сохранение состояния при перезагрузке.
- **Data Persistence**: Все данные синхронизируются с `localStorage` (эмуляция БД на стороне клиента).

---

## 🛠 2. Критические задачи (Backlog)

### 🔥 Этап 1: Инфраструктура и Безопасность
- **Backend Migration**: Перенос AI-логики (Gemini) на серверный слой (Node.js/Express).
- **Environment Secrets**: Вынос `API_KEY` в серверные переменные окружения.
- **Auth Service**: Интеграция Firebase или Supabase Auth для реальной регистрации пользователей.

### 📈 Этап 2: SEO и Масштабирование
- **SSR/SSG**: Переход на Next.js для серверного рендеринга статей. Сейчас SEO реализовано через динамическую подмену `Head`, что не оптимально для Яндекса.
- **Sitemap Generator**: Автоматическая генерация карты сайта на основе массива `BLOG_ARTICLES`.

### 💰 Этап 3: Финансы и Оплаты
- **Webhooks**: Настройка приема уведомлений от платежных систем (ЮKassa / Stripe).
- **Invoicing**: Генерация PDF-счетов для B2B заказов.
