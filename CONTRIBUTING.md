# Руководство по внесению изменений (Contributing)

Мы приветствуем вклад в развитие TextFlow! Пожалуйста, следуйте этим правилам, чтобы процесс разработки был эффективным и приятным.

## Как внести вклад

1.  **Форкните репозиторий**.
2.  **Создайте ветку для вашей фичи** (`git checkout -b feature/AmazingFeature`).
3.  **Зафиксируйте изменения** (`git commit -m 'Add some AmazingFeature'`).
4.  **Отправьте ветку в GitHub** (`git push origin feature/AmazingFeature`).
5.  **Откройте Pull Request**.

## Правила написания кода

- **TypeScript**: Мы используем строгую типизацию. Избегайте использования `any`. Все новые интерфейсы должны быть описаны в `src/types.ts`.
- **Компоненты**: Используйте функциональные компоненты и хуки. Разделяйте логику и представление.
- **Стили**: Используйте только Tailwind CSS классы. Избегайте инлайновых стилей и кастомного CSS, если это возможно.
- **Именование**: Используйте `PascalCase` для компонентов и `camelCase` для функций и переменных.

## Проверка кода

Перед отправкой PR убедитесь, что:
1. Проект собирается без ошибок: `npm run build`.
2. Отсутствуют ошибки типизации: `npm run lint`.
3. Код отформатирован и легко читается.

## Сообщения об ошибках

Если вы нашли баг, пожалуйста, создайте Issue с подробным описанием:
- Шаги для воспроизведения.
- Ожидаемое поведение.
- Скриншоты (если применимо).

Спасибо, что помогаете делать TextFlow лучше!
